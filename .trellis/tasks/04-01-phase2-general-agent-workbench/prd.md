# Phase 2 General Agent Workbench

## Goal

Turn the shell into a general-purpose desktop agent workbench instead of a set of specialized workflow screens.

## Scope

- Session UX for broad goals
- Workspace and retrieval visibility inside the agent flow
- Tool and MCP capability visibility
- Agent progress and plan visibility

## Requirements

- The UI should support broad knowledge-work goals without assuming a narrow vertical use case.
- Workspace content and tool access should feel integrated into the main session experience.
- Users should be able to understand what the agent is doing, what it plans to do next, and what context it is using.
- Product copy and navigation should avoid predefined-workflow-first framing.

## Acceptance Criteria

- [ ] A user can use the system for a general desktop goal without encountering predefined-workflow-centric UI.
- [ ] Workspace context is visible and usable from the main agent experience.
- [ ] Tool/MCP capability visibility is present where it helps the user supervise the agent.
- [ ] Session and run views feel like one product, not separate products.

## Out Of Scope

- Domain-specific workflow builders
- Heavy visual redesign detached from runtime clarity
