# Phase 4 Packaging And User Docs

## Goal

Make IronCowork distributable as a desktop app with user-facing setup and usage docs that match the session-first autonomous-agent direction.

## Scope

- Tauri packaging targets
- Release asset polish
- Real icons and bundle metadata
- User-facing setup docs
- Usage docs for sessions, approvals, workspace, and browser-vs-desktop mode

## Requirements

- Define supported packaging targets and verification steps.
- Replace placeholder icons and temporary bundle metadata.
- Document user setup for browser mode, desktop mode, API keys, and storage path expectations.
- Document the session-first model, Ask/Yolo, workspace usage, and explicit non-goals such as built-in remote exposure.

## Acceptance Criteria

- [ ] Supported desktop packaging targets can be built and checked.
- [ ] Release assets reflect IronCowork branding rather than placeholders.
- [ ] User docs cover installation, setup, and core agent usage.
- [ ] Packaging and documentation checks are part of release readiness.

## Out Of Scope

- Marketing site work
- Non-local hosted deployment features
